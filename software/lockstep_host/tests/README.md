# tests

本目录保存自动化测试代码和测试夹具。优先覆盖风险较高的核心规则：

- 研发/测试子工作区互不混写。
- 任务产物路径位于当前任务目录内。
- 回读失败不得运行。
- 未修复阻断错误不得生成 `pass`。
- 资源路径只以逻辑 ID、版本和摘要进入任务与报告。
