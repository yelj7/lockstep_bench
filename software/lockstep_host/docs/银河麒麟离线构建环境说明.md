# 银河麒麟离线构建环境说明

构建环境套件只用于编译机，不进入最终上位机安装包。目标平台固定为 `Desktop-V10-SP1-General-Release x86_64`。

## 检查当前环境

```bash
cd software/lockstep_host
./scripts/check-kylin-build-env.sh
```

检查脚本按 `packaging/kylin-env/build-requirements.psv` 输出 `[OK]`、`[MISSING]` 或 `[WRONG VERSION]`。除 `qmake` 版本外，还会通过 `pkg-config` 验证 Qt Core、Gui、Widgets、Network、SerialPort 和 hidapi-hidraw 开发模块。生成环境锁文件：

```bash
./scripts/check-kylin-build-env.sh --write-lock build-environment.lock.json
```

`build-environment.lock.json` 记录需求清单摘要，以及每个工具/库的探测方式、版本约束和实际值，用于比较两次构建环境是否一致。

## 制作离线套件

在能够访问银河麒麟 V10 SP1 对应内网镜像的同架构参考机上，将需求清单根包及全部传递依赖下载到一个独立目录。不得混入其他发行版或 ARM64 软件包。然后执行：

```bash
./scripts/prepare-kylin-build-env-bundle.sh \
  --deb-dir /path/to/complete-debs \
  --output /path/to/kylin-build-environment-amd64
```

制作脚本会：

- 验证需求清单中的全部根包及 CMake/Qt/hidapi 版本，拒绝同名多版本包、ARM64/其他架构包和已有输出目录；
- 仅使用临时 `file:` 软件源和空 `dpkg status` 执行 `apt-get -s --reinstall`，不允许参考机已安装的包掩盖仓库中缺失的传递依赖；
- 生成 `deb-packages.psv`，锁定每个 DEB 的包名、版本、架构、文件名和 SHA-256；
- 生成本地 apt 仓库及强制校验的 `SHA256SUMS`；
- 全部步骤成功后才原子生成输出目录，失败时清理临时目录。

脚本会强制校验制作机为 `Desktop-V10-SP1-General-Release x86_64`。空状态依赖预演意味着输入 DEB 目录必须真正包含根包的全部传递依赖，仓库不完整时不会生成套件。

## 离线一键部署

将整个 `kylin-build-environment-amd64` 目录复制到离线麒麟编译机，执行：

```bash
cd kylin-build-environment-amd64
./bootstrap-kylin-build-env.sh --bundle "$PWD"
```

部署脚本启动后首先强制校验 `SHA256SUMS`，在通过前不会执行套件内的检查脚本。随后只配置临时 `file:` 软件源，不读取系统网络软件源。环境已经满足时不会调用 apt；存在缺失时只对缺失或版本错误项对应的 DEB 发起离线安装，不会主动重装或升级已满足要求的 `dpkg`、`libc-bin`、`coreutils` 等基础包。

真实安装前还会执行一次 `apt-get -s`。如果目标机基线不同、传递依赖缺失或冲突，脚本会在修改系统前退出。安装完成后强制复检并生成 `build-environment.lock.json`。

`sha256sum`、`apt-get`、Bash 和基本 Debian 包管理能力视为银河麒麟系统基础前提。其中 `sha256sum` 不能由本脚本自举安装，因为在信任任何待安装 DEB 之前必须先校验套件完整性。

部署完成后回到源码目录运行：

```bash
./scripts/build-kylin-offline-package.sh
```
