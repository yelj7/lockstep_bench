# 模块开发完成说明

日期：2026-07-08

本文简要记录本轮已完成的底层 C++ 模块、主要接口和功能范围。源码均放在 `software/lockstep_host/src` 下已有对应模块目录中，未新增通用源码目录。

## workspace

源码：`src/workspace/workspace_manager.h/.cpp`

主要接口：
- `WorkspaceManager::createWorkspace/openWorkspace/switchMode`
- `WorkspaceManager::createTask/loadTask/listTasks/renameTask`
- `WorkspaceManager::saveTaskInputs/importTaskInputFile/handleInputChange`
- `WorkspaceManager::attachArtifact/getTaskPaths`

功能：
- 创建和打开研发/测试工作区，初始化 `workspace_index.json`、`mode_config.json`、`system_logs`、`tasks` 等目录与索引。
- 创建任务、生成任务 ID、校验同一子工作区内任务名称唯一性。
- 导入任务输入文件并记录任务内相对路径、文件名、摘要和大小，不保存外部原始路径。
- 输入变更时支持覆盖当前任务、创建新任务或取消；覆盖会清理旧日志、证据、波形、报告等任务产物。
- 维护任务级产物索引 `evidence/artifacts.json`。

## resources

源码：`src/resources/resource_manager.h/.cpp`

主要接口：
- `ResourceManager::validateResourcePack`
- `ResourceManager::getModeResourceSnapshot`
- `ResourceManager::resolveBoardProfile`

功能：
- 解析并校验 `resources/manifest.json`。
- 支持资源状态 `enabled/placeholder/warning/missing/invalid/disabled`。
- 读取板卡 profile 字段，包括 `host`、`gdbPort`、`tclPort`、`jtagKhz`、`ramBaseAddress`、`defaultRunAddress`、`resetStrategy` 等。
- 输出给任务和报告的资源快照只包含资源 ID、版本、摘要和状态，不泄漏固化资源的物理路径。
- 对资源文件存在性和 sha256 摘要进行校验。
- 新增 `lockstep_trace_profiles` 资源组，默认绑定 `trace.noelv.lockstep_512`。

## target_control

源码：`src/target_control/target_control.h/.cpp`

主要接口：
- `DebugAccess` 抽象调试访问接口
- `TargetConnectionService::connectTarget/disconnectTarget/runPrecheck`
- `ProgramController::detectImage/programTarget/verifyReadback/runTarget`

功能：
- 提供连接、断开、状态、读、写、复位、运行、中止等统一调试访问抽象。
- 支持 ELF 魔数识别、SREC/S19/S28/S37/MOT、Intel HEX/HEX/IHEX/IHX 解析、BIN 后缀识别。
- 校验镜像写入范围，越界或无法识别时阻断烧写。
- 执行烧写、回读比较和运行控制闭环。
- 回读校验未通过时 `runTarget` 返回 `NotAllowed`，不允许运行。

## reporting

源码：`src/reporting/report_generator.h/.cpp`

主要接口：
- `ReportGenerator::calculateConclusion`
- `ReportGenerator::generateReport`

功能：
- 生成 schema 2.0 的版本化 `report.json`、自包含 `report.html`、`manifest.json` 和 latest 入口。
- `pass` 强制条件为：烧写成功记录、回读校验通过记录、运行控制真实返回或快照。
- VCD 波形、协议解析、采集、故障注入和 warning 只作为记录，不阻断通过结论。
- 存在未修复阻断错误时结论为 `blocked`；强制证据缺失时结论为 `incomplete`。

## error_handling

源码：`src/error_handling/error_registry.h/.cpp`

主要接口：
- `ErrorRegistry::createRecord`
- `ErrorRegistry::appendTaskError/appendSystemError`
- `ErrorRegistry::appendLog`
- `ErrorRegistry::loadTaskErrors/writeTaskErrors/resolveTaskError`
- `ErrorRegistry::unresolvedBlockingErrors/canContinueWith/canAcknowledgeWithoutResolution`
- `ErrorRegistry::runMaintenanceCheck`

功能：
- 定义统一错误记录、严重级别、状态和日志入口模型。
- 支持灵活错误码字符串，不强制固定编码格式。
- 任务错误写入 `logs/error_events.json` 和 `logs/task_log.jsonl`。
- 系统错误写入 `system_error_events.json` 和 `system_log.jsonl`。
- `warning/info` 可确认后继续；`error/blocking/critical` 未修复时不能忽略。
- 提供维护检查结果，用于汇总日志目录状态和未修复错误状态。
- 新增 `appendTaskErrorsBulk`，支持 M12 一次解析后批量写入聚合诊断。

## waveform_viewer

源码：`src/waveform_viewer/waveform_trace_viewer.h/.cpp`

主要接口：
- `WaveformTraceViewer::loadTask`
- `fixedWaveformRelativePath/fixedTraceSchemaRelativePath/fixedTraceAnalysisRelativePath`

功能：
- 固定读取当前任务 `waveform/lockstep_trace.vcd`、`lockstep_trace_schema.json`、`lockstep_trace_analysis.json`。
- 读取 M12 analysis，整理 9 组协议展示模型、关键行为和诊断摘要。
- 检测 analysis 与当前 VCD 摘要是否不一致，并提示重新解析。
- UI 页面已接入当前任务固定波形状态、9 组树形结果、关键行为和诊断显示；进入波形/协议页时若存在固定 VCD 且 analysis 缺失或过期，会自动调用 M12 重新解析。

## protocol_analyzer

源码：`src/protocol_analyzer/protocol_analysis.h/.cpp`

主要接口：
- `ProtocolAnalyzer::analyzeTask`
- `ProtocolAnalysisRequest/ProtocolAnalysisResult`

功能：
- 校验固定 VCD 主信号 `lockstep_trace_sample[511:0]`。
- 解析 VCD 时间戳和 512-bit packed 采样值。
- 生成 `waveform/lockstep_trace_analysis.json`，默认包含 AHB、UART、SPI、CAN、I2C、ETH、USB、JTAG、mismatch 九组。
- 完整解析 `lockstep_trace_sample[506:502]` 的 `mismatch[4:0]`，多 bit mismatch 合并为复合事件并保留全部 items。
- 对未固化字段映射的 8 组协议输出 `not_captured` 和聚合诊断，不生成伪事务。
- 聚合诊断和 VCD 合同错误会通过 M14 批量上报，并在 analysis 中保存 `error_id`。
