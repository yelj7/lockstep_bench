# error_handling

本目录对应 M14，负责统一错误事件、错误提示映射、任务日志、系统诊断日志和维护检查。

边界规则：

- 业务模块在错误源附近产生结构化错误事件。
- 本模块负责解码、记录、展示建议和查询未修复阻断错误。
- 不替业务模块修复错误，不把阻断错误降级为成功。
- `warning` 允许继续，但必须写入日志和 `report.json`。
