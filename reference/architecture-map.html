锁步上位机架构速查

阅读任何功能时，都先判断它位于哪一层，再追踪输入、状态、输出和失败证据。

UI 控件→UiActionRequest→WorkbenchController
领域模块→状态与规则→任务产物

三层职责

  • 界面层：ui_common/，展示状态、收集输入、发出动作，不直接操作硬件或任务文件。
  • 适配层：apps/workbench_controller.*，协调模块、维护当前会话状态、把结果刷新到界面。
  • 领域层：workspace/ 等模块，执行规则、产生记录和结构化产物。

领域模块

  • workspace：工作区、任务、输入和产物索引。
  • resources：固化资源、板卡 profile 和模式配置。
  • workflow：流程状态、阶段 gate 和失败阻断。
  • target_control：连接、预检、烧写、回读、运行和停止。
  • fault_injection / acquisition：注错与采集。
  • protocol_analyzer / waveform_viewer：协议分析与波形展示。
  • reporting / error_handling：报告、结论、日志与错误证据。

固定追踪问题

  1. 谁产生输入？输入的数据类型是什么？
  2. 哪个对象拥有状态？状态转换条件是什么？
  3. 输出写到 UI、内存还是任务目录？
  4. 失败如何阻断后续操作并留下证据？
