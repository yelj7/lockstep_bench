C++ 与 Qt 读码速查

只覆盖当前仓库读码所需含义。遇到这些写法时，先翻译职责和数据流，不必立刻掌握语言全部细节。

C++ 形状

代码形状当前应如何读
Type object;创建一个类型为 Type 的对象。
Type object(args);调用构造函数，用参数创建对象。
Type* pointer保存另一个对象的地址；可能为空。
&object取得对象地址，交给需要指针的调用方。
pointer->method()通过指针调用对象方法。
object.method()直接调用对象方法。
Namespace::Type命名空间中的类型，避免同名冲突。
Class::method属于某个类的方法。
const Type& value只读引用：不复制对象，也不通过它修改对象。
nullptr明确表示“当前没有对象”。
enum class一组受类型约束的命名选项或状态。

Qt 事件形状

代码形状当前应如何读
Q_OBJECT该类启用 Qt 元对象能力，包括信号槽。
signals:声明对象能够发布的事件。
emit signal(data)发布事件，并携带数据。
connect(sender, signal, receiver, handler)注册规则：信号发生时执行处理器。
[this](...) { ... }匿名函数；捕获当前对象，以便调用成员方法。
application.exec()进入事件循环，持续等待按钮、定时器、串口等事件。
new Widget(parent)创建 Qt 对象并指定父对象；父对象通常负责销毁它。

本仓库固定翻译

emit actionRequested(request)
        ↓
“MainWindowShell 发布了一次带参数的 UI 动作请求”

connect(window_, &MainWindowShell::actionRequested,
        this, [this](const UiActionRequest& request) {
            handleAction(request);
        });
        ↓
“窗口发布动作时，由当前控制器调用 handleAction”

主资料：Qt 官方 Signals & Slots，用于核对信号槽的正式语义。
