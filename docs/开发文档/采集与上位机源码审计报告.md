# 采集与上位机源码审计报告

## 1. 审计边界与方法

- 硬件边界为 `rtl/lockstep_capture/lockstep_capture_sources.lst` 的 22 个生产 RTL，并向上核对 FT601 板级入口、向下核对实例和端口消费者；不审阅锁步核内部实现。
- 软件边界为 `software/lockstep_host/src` 的 45 个 `.h/.cpp` 生产源码及 12 个 CMake 入口，结合链接目标、include/call 引用、Workbench 工作流和 13 项 CTest 检查可达性。
- 删除只接受“无调用/无消费者、与现有状态机重复、或会破坏当前合同”的证据。工作树起始即存在的未提交删除单独标记，不冒充为本轮新删除。
- RTL 结论已通过 Vivado 2022.2 XSIM、综合、实现后 timing/route/DRC/CDC/bus-skew/I/O 自动门禁；Host 结论已通过独立临时构建目录的全量 CTest。

## 4. 删除证据

### 4.1 本轮确认删除

| 文件/符号 | 删除依据 |
|---|---|
| `workflow/workflow_engine.h/.cpp` | `WorkflowEngine` 仅记录阶段、没有反向驱动 UI 或硬件 gate，与 Controller 的真实操作顺序重复，属于只写状态机。 |
| `workflow/CMakeLists.txt`、`workflow/README.md` | 源码删除后 target 无消费者，文档描述的 gate 已由 Controller/target/acquisition 承担。 |
| `workbench_controller.*` 的 `workflow_`、`flowState_`、`flowMode()`、`recordStageResult/startFlow` | 全部只有写入，无读取、展示或持久化消费者。 |
| `workbench_controller.h` 的常驻 `connectionService_` | 服务无跨调用状态，改用局部对象。 |
| `workbench_controller.cpp` 的 `imageToJson()` | 仅服务已删除 workflow detail，无剩余调用。 |
| `lockstep_protocol_probe_pack.v` 的 `unused_sample_index_w` | 自赋值且无扇出，仅为压制未使用端口。 |
| `lockstep_ft601_external_sample_top.v` 的本地 ARM-relative index 计数链 | 会覆盖真实 `sample_abs_index_i` 并造成 VCD/事件时间轴错位。 |
| `lockstep_event_capture_controller.v` 的自由运行 timestamp | 无消费者且与统一绝对时间源重复。 |
| `lockstep_global_timestamp.v` 的旧自由运行实现 | 周期计数不等于有效样本索引，改为 32-bit 绝对索引的 64-bit wrap 扩展。 |
| `lockstep_event_frame_source.v` 的 `effective_end_reason_w` | overflow 覆盖主结束原因会丢失真实终止原因，现由独立字段报告。 |
| `lockstep_capture_protocol_stream_core.v` 的固定 timeout 参数和死统计连线 | 与 EVENT_CONFIG 重复或无消费者。 |
| 同文件的固定三拍 FIFO settle | 拍数猜测不能证明排空，改为含预取状态的显式 `read_empty`。 |
| 同文件旧 STOP/timeout/FIFO reset CDC 结构 | 分别替换为 toggle-to-pulse、两级同步加 trigger 快照、双域 reset-ready。 |
| `lockstep_ft601_external_sample_top.v` 的全向量复位比较 | 形成 CDC-7/组合释放风险，改为标准 ASYNC_REG 移位复位链。 |

## 5. 审计结论

- 生产采集 RTL 22/22、Host `.h/.cpp` 45/45 和构建入口均已形成可达性记录。
- 精简集中在无消费者接口、重复状态机、错误局部时间轴、猜测式排空和死 CDC 连线；没有为追求行数删除板级入口、协议边界或错误上报路径。
- 最终证据为 Host 13/13 CTest、远端 XSIM 17/17、e416 debug/release 自动实现门禁和 SHA-256 全通过；c477 完整数据面板测及 e416 最终复烧、FT601 STATUS/CONFIG/ARM/STOP 冒烟均通过。
- 当前不扩展真实 `program_done`、SPI sparse 双边沿、JTAG 指令级边界扫描、CAN 外部链路、ETH/USB；这些缺口只记录，不伪造完成。
