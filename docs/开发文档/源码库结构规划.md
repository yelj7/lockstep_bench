## 当前整理决策

- 保留现有 `src/apps/` 和 `src/ui_common/` 代码，不拆分已经可运行的 UI 预览骨架。
- 新增 `src/CMakeLists.txt`，统一管理源码模块子目录。
- 允许相近模块合并，但目录名必须表达稳定领域边界；不使用 `core/` 这类过度概括目录。
- `target_control/` 合并目标连接、自研片上调试访问、程序烧写回读与运行控制，内部以 `connection`、`debug_access`、`programming` 三个子边界区分。
- `acquisition/` 合并采集控制和帧流处理，内部以 `control`、`frames` 两个子边界区分。
- 为尚未实现的业务目录创建 CMake interface 目标和目录说明，先固定边界，不提前伪造实现。
- 新增 `.gitignore`，忽略本地 CMake 构建目录和编译产物；不删除本地已有 `build-*` 目录。
- `profiles/`、`scripts/`、`tests/`、`examples/`、`software/lockstep_host/docs/` 先以 README 固化边界，后续按模块进度逐步填充。

## 任务级运行数据结构口径

后续 M02/M13/M14 实现以如下任务级结构为准：

```text
workspace/
  research_workspace/
    workspace_index.json
    mode_config.json
    system_logs/
    tasks/
      <task_id>/
        task.json
        inputs/
        logs/
        evidence/
        waveform/
        reports/
  test_workspace/
    workspace_index.json
    mode_config.json
    system_logs/
    tasks/
      <task_id>/
        task.json
        inputs/
        fault_injection/
        logs/
        evidence/
        waveform/
        reports/
```

强制规则：

- 研发模式只读写 `research_workspace/`，测试模式只读写 `test_workspace/`。
- 任务和报告不保存导入前外部路径、profile 物理路径、模板物理路径或临时后端路径。
- 回读失败、回读不一致或缺少回读通过记录时，不允许运行。
- 未修复阻断错误不得继续依赖阶段，也不得生成 `pass`。
- `warning`、VCD、协议解析、采集文件和故障注入记录可以进入报告记录，但 v1 不作为 `pass` 强制证据。
