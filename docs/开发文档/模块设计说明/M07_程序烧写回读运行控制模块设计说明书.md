# 1. 设计定位

M07 在目标已连接并通过预检后，完成程序镜像自动识别、写入范围解析、目标写入、回读比较、启动/复位/中止运行控制和运行状态记录。M07 产出的烧写成功记录、回读校验通过记录、运行控制真实返回或快照共同构成 `pass` 的强制事实基础。

# 2. 设计输入与约束

输入：

- 当前任务和任务内程序镜像副本。
- M03/M05 提供的 profile 摘要、`ramBaseAddress`、`defaultRunAddress`、运行策略等。
- M06 调试访问接口。

镜像格式裁决：

- 支持 ELF。
- 支持 SREC 及常见 Motorola S-record 后缀：`.srec`、`.s19`、`.s28`、`.s37`、`.mot`。
- 支持 02 设计书提到的三类镜像：带段信息的可执行镜像、原始二进制镜像、带地址记录的文本镜像。
- 原始二进制镜像支持 `.bin`。
- 带地址记录的文本镜像包括 Intel HEX，可支持 `.hex`、`.ihex`、`.ihx`。
- 自动检测格式；不在支持范围内或无法识别则报错。

约束：

- 未连接或预检失败时不得烧写、回读、运行。
- 镜像无效或写入范围越界时不得烧写。
- 回读失败、回读不一致或缺少回读记录时不允许运行。
- 程序烧录页不提供格式、BIN 地址、Offset、运行地址覆盖等手动修改入口；这些值来自镜像解析、任务内程序副本和固化 profile。
- 程序运行页只读取当前任务已确定的程序镜像名称和路径，不提供重新选择程序镜像入口。
- 进度条表示控制阶段进度，不等同于已写入或已回读字节比例。
- warning 允许记录，不阻断 pass。

# 3. 模块职责与不负责事项

职责：

- 校验镜像存在性、可读性和格式。
- 自动识别镜像类型。
- 解析写入范围和入口地址。
- 校验写入范围是否落在 profile 允许范围内。
- 调用 M06 写入目标。
- 调用 M06 回读相同范围并比较。
- 调用 M06 执行复位、运行、中止和状态查询。
- 生成烧写、回读、运行控制记录。
- 向 UI 输出烧写、回读、运行、中止的阶段进度和摘要文本。

不负责：

| 事项 | 归属 |
|---|---|
| 任务创建和镜像导入 | M02 |
| profile schema | M03 |
| 连接预检 | M05 |
| 调试命令后端 | M06 |
| VCD/协议解析 | M09-M12 |
| 报告结论 | M13 |

# 4. 数据对象与状态模型

| 对象 | 说明 |
|---|---|
| `ProgramImageManifest` | 原始文件名、任务内路径、摘要、大小、自动识别类型 |
| `ImageSegment` | 写入地址、长度、数据摘要 |
| `WriteRecord` | 写入范围、写入结果、原始返回 |
| `ReadbackVerifyRecord` | 期望长度、实际长度、差异数量、校验状态 |
| `RunControlRecord` | 操作、入口地址、真实返回或快照 |
| `OperationProgress` | 操作类型、阶段、百分比、阶段说明、是否可取消 |

镜像类型：

| 类型 | 后缀/检测 |
|---|---|
| `elf` | ELF magic 或 `.elf` |
| `srec` | S-record 记录或 `.srec/.s19/.s28/.s37/.mot` |
| `bin` | `.bin` 且需有 profile 或任务提供基地址 |
| `intel_hex` | Intel HEX 记录或 `.hex/.ihex/.ihx` |

回读状态：`not_run`、`passed`、`mismatch`、`failed`。

运行状态：`not_allowed`、`ready`、`running`、`halted`、`aborted`、`failed`。

阶段进度模型：

| 操作 | 阶段 |
|---|---|
| 烧写 | `check_debug_access`、`detect_image`、`parse_write_plan`、`write_segments`、`confirm_write_result`、`persist_write_record` |
| 回读 | `check_readback_access`、`prepare_read_ranges`、`read_segments`、`compare_data`、`persist_verify_record` |
| 运行 | `check_run_gate`、`dispatch_run`、`capture_run_status`、`persist_run_record` |
| 中止 | `check_halt_access`、`dispatch_halt`、`capture_halt_status`、`persist_halt_record` |

# 5. 对外接口与调用关系

| 接口 | 调用方 | 输出 |

| `programTarget(task_id)` | M04/UI | 烧写记录 |
| `verifyReadback(task_id)` | M04/UI | 回读校验记录 |

| `getOperationProgress(task_id, operation)` | UI | 当前阶段进度 |

# 6. 主流程与异常流程

镜像识别：

1. 读取任务内镜像副本。
2. 先按文件 magic/内容特征识别，再用后缀辅助确认。
3. 识别失败则生成不支持格式错误。
4. 对 ELF/SREC 解析地址段，对 BIN 使用 profile 或任务配置基地址。

烧写：

1. 输出 `check_debug_access`，检查连接和预检 gate。
2. 输出 `detect_image`，检查镜像类型。
3. 输出 `parse_write_plan`，解析写入范围和地址约束。
4. 输出 `write_segments`，按段调用 M06 写入。
5. 输出 `confirm_write_result`，确认写入真实返回。
6. 输出 `persist_write_record`，生成烧写成功或失败记录。

回读：

1. 输出 `check_readback_access`，仅在烧写成功后执行。
2. 输出 `prepare_read_ranges`，准备与写入一致的目标范围。
3. 输出 `read_segments`，对相同目标范围调用 M06 读取。
4. 输出 `compare_data`，比较期望数据和实际数据。
5. 输出 `persist_verify_record`，记录 `expected_length`、`actual_length`、`diff_count`。
6. 长度一致且 `diff_count=0` 才是 `passed`。

运行控制：

1. 输出 `check_run_gate`，仅当烧写成功且回读通过时允许运行。
2. 输出 `dispatch_run`，根据入口地址调用 M06 `run`。
3. 输出 `capture_run_status`，保存运行控制真实返回或状态快照。
4. 输出 `persist_run_record`，生成运行记录。
5. 无真实返回或快照时不得作为 pass 证据。

中止控制：

1. 输出 `check_halt_access`，检查目标连接和运行控制能力。
2. 输出 `dispatch_halt`，调用 M06 `halt`。
3. 输出 `capture_halt_status`，保存中止真实返回或状态快照。
4. 输出 `persist_halt_record`，生成中止记录。

异常流程：

| 异常 | 处理 |
|---|---|
| 不支持后缀或内容不可识别 | 报错并阻断烧写 |
| 地址越界 | 报错并阻断烧写 |
| 写入失败 | 阻断回读通过和运行 |
| 回读不一致 | 阻断运行 |
| 回读失败 | 阻断运行 |
| 运行真实返回缺失 | 运行阶段不通过 |
| 中止真实返回缺失 | 中止结果显示为失败或未知，不伪造成成功 |

```text
  logs/program_write.log
  logs/program_run.log
```

# 8. 错误处理、日志与证据规则

错误事件字段包括：`code`、`source_module=M07_PROGRAM_WRITE_VERIFY_RUN`、`failure_stage`、`task_id`、`program_image_id`、`expected_length`、`actual_length`、`diff_count`、`raw_return_artifact_id`。

pass 强制证据：

1. 烧写成功记录。
2. 回读校验通过记录。
3. 运行控制真实返回或快照。

VCD 波形和协议解析结果仅作为记录，不作为 pass 阻塞条件。

# 9. 与 UI 的数据合同

程序烧录页显示：

- 镜像文件名、任务内镜像路径、自动识别类型、摘要。
- 写入范围和入口地址摘要；不提供手动修改入口。
- 烧写状态。
- 回读状态、差异数量。
- 烧写进度条和回读进度条，按阶段推进。
- 烧录与回读摘要，以及“清空窗口”按钮。

程序运行页显示：

- 当前任务已确定的镜像名称和路径，只读显示，不提供选择按钮。
- “程序运行”“程序中止”两个控制按钮。
- 运行进度条和中止进度条，按阶段推进。
- 运行摘要输出区域，不再提供回读/运行摘要切换。

UI 行为：

- 不支持格式直接报错。
- 回读失败或不一致时程序运行页的运行入口不可用。
- 不提供强制运行入口。
- 程序烧录页不包含程序运行、程序中止、运行摘要。
- 程序运行页不重新选择镜像，必须使用程序烧录流程确定的任务内镜像。

# 10. 测试场景与验收标准

| 场景 | 验收标准 |
|---|---|
| ELF 镜像 | 自动识别并解析段信息 |
| SREC/S19/S28/S37/MOT | 自动识别并解析地址记录 |
| BIN 镜像 | 使用基地址形成写入范围 |
| 未知后缀 | 报错并阻断 |
| 地址越界 | 报错并阻断 |
| 烧写阶段推进 | 从调试访问可用性检查推进到写入结果确认 |
| 回读阶段推进 | 从回读可用性检查推进到比较结果确认 |
| 回读一致 | 运行可用 |
| 回读失败 | 运行不可用 |
| 运行成功 | 保存真实返回或快照 |
| 中止成功 | 保存真实返回或快照 |
| 程序运行页重新选择镜像 | UI 不提供该入口 |

# 11. 设计假设

1. BIN 的基地址来自 profile 的 `ramBaseAddress` 或固化配置，UI 不提供手动输入。
2. 多段镜像可先形成一条汇总写入记录，内部保留段明细。
3. 回读差异 v1 按字节统计。
4. ELF 目标实现应按可加载段解析；若开发早期临时按整文件写入，只能用于联调，不得作为真实上板验收结论。
