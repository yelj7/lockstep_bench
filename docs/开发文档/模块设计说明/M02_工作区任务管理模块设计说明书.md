# 1. 设计定位

M02 是上位机软件的数据边界基础模块，负责项目工作区、研发/测试子工作区、任务生命周期、任务级目录、任务输入和任务产物挂接规则。它为 UI、流程编排、归档报告、错误日志和各业务执行模块提供统一的任务上下文。

本模块只设计任务级管理。所有日志、证据、波形、协议解析结果和报告均直接归属当前任务目录。

# 2. 设计输入与约束

设计输入：

- `01_锁步研发测试系统上位机需求规格说明书.md`
- `02_锁步研发测试系统上位机软件设计说明书.md`
- `03_锁步研发测试系统上位机软件数据库设计说明书.md`
- `04_锁步研发测试系统上位机软件开发计划.md`
- `锁步研发测试系统上位机开发任务进度确认表.csv`
- `模块设计说明/UI设计说明书.md`
- 用户裁决：产物直接归属任务；报告通过只强制依赖程序跑通所需证据

关键约束：

| 约束 | 设计口径 |
|---|---|
| 模式隔离 | 研发模式只读写 `research_workspace/`，测试模式只读写 `test_workspace/` |
| UI 边界 | 主界面不设历史、资源或系统状态入口；顶部状态栏不显示工作区路径 |
| 任务目录 | 所有任务产物直接落在 `tasks/<task_id>/` 下的固定子目录 |
| 任务 ID | 系统生成、用户可见、不可编辑 |
| 任务名称 | 当前子工作区内唯一；去首尾空格后大小写不敏感比较 |
| 关键输入变更 | 必须询问覆盖当前任务或新建任务，不得静默沿用旧证据 |
| 外部路径 | 任务输入和报告不得保存导入前外部路径 |

# 3. 模块职责与不负责事项

模块职责：

- 创建或加载项目工作区。
- 初始化 `research_workspace/` 与 `test_workspace/`。
- 初始化 `workspace_index.json`、`mode_config.json`、`system_logs/` 和 `tasks/`。
- 创建、加载、保存、重命名任务。
- 生成任务 ID，维护任务名称唯一性。
- 保存程序镜像清单、采样配置；测试模式可保存故障注入配置。
- 处理关键输入变更并清理旧产物。
- 提供日志、证据、波形、协议解析和报告的任务级挂接路径。
- 向 UI 提供当前模式、当前任务、流程阶段、保存状态和最近错误来源。

不负责事项：

| 不负责事项 | 归属模块 |
|---|---|
| UI 布局和主导航渲染 | M01 |
| 固化资源校验与默认模板绑定 | M03 |
| 流程状态机与按钮前置规则 | M04 |
| 目标连接和预检 | M05 |
| 调试访问后端 | M06 |
| 烧写、回读、运行控制 | M07 |
| 报告结论计算 | M13 |
| 错误码解码和日志格式化 | M14 |

# 4. 数据对象与状态模型

核心对象：

| 对象 | 说明 |
|---|---|
| `ProjectWorkspace` | 用户选择的项目工作区根目录 |
| `SubWorkspace` | `research_workspace` 或 `test_workspace` |
| `Task` | 当前模式下的研发任务或测试任务 |
| `TaskInputSet` | 程序、采样、测试故障注入等关键输入 |
| `ArtifactIndex` | 当前任务内产物索引 |

推荐任务 ID：

```text
t_<mode_code>_<yyyyMMddHHmmss>_<seq4>
```

任务状态：

| 状态 | 含义 |
|---|---|
| `draft` | 已创建但关键输入不完整 |
| `ready` | 关键输入已保存 |
| `running` | 当前任务正在执行受控流程 |
| `completed` | 当前任务流程结束并已形成记录 |
| `failed` | 当前任务存在失败或阻断错误 |
| `aborted` | 用户中止 |
| `stale` | 关键输入已变更，旧产物不可作为当前结论依据 |

任务输入摘要字段：

| 字段 | 说明 |
|---|---|
| `program_manifest_id` | 当前任务程序镜像清单 |
| `sampling_config_id` | 采样配置 |
| `fault_injection_config_id` | 测试模式故障注入配置，可为空 |
| `resource_snapshot` | 当前模式固化资源 ID、版本和摘要 |

# 5. 对外接口与调用关系

| 接口 | 调用方 | 输出 |
|---|---|---|
| `openWorkspace(path)` | 启动页/UI | 工作区加载结果 |
| `createWorkspace(path)` | 启动页/UI | 初始化后的项目工作区 |
| `switchMode(mode)` | UI | 当前子工作区上下文 |
| `listTasks(mode, filter)` | UI | 任务列表 |
| `createTask(mode, template)` | UI/M04 | 任务 ID 和任务目录 |
| `loadTask(task_id)` | UI/M04 | 任务上下文 |
| `saveTaskInputs(task_id, inputs)` | UI/M04 | 输入摘要和保存状态 |
| `handleInputChange(task_id, changed_inputs, action)` | UI/M04 | 任务状态和清理结果 |
| `attachArtifact(task_id, artifact)` | 各业务模块/M13 | 产物索引记录 |
| `getTaskPaths(task_id)` | 各业务模块 | 任务级相对目录集合 |

调用关系：M01 通过 M02 获取当前任务上下文；M04 通过 M02 检查任务输入和产物状态；M13 通过 M02 写入报告索引；M14 通过 M02 获取任务级日志路径。

# 6. 主流程与异常流程

工作区加载流程：

1. 用户选择项目工作区。
2. M02 校验目录可读写。
3. 若缺少 `research_workspace/` 或 `test_workspace/`，按初始化规则创建。
4. 读取工作区索引和模式配置。
5. 校验索引中的任务目录存在性。
6. 返回当前可用模式和任务摘要。

任务创建流程：

1. UI 请求新建空任务或复制现有任务配置。
2. M02 生成任务 ID 和默认任务名称。
3. 创建 `tasks/<task_id>/` 目录。
4. 初始化 `task.json`、`inputs/`、`logs/`、`evidence/`、`waveform/`、`reports/`。
5. 写入任务索引。

输入变更流程：

1. 检测程序镜像、采样配置或测试故障注入配置变更。
2. UI 要求用户选择覆盖当前任务、新建任务或取消。
3. 覆盖当前任务时，只替换发生变化的输入文件。
4. 清理与旧输入绑定的 `logs/`、`evidence/`、`waveform/` 和 `reports/`。
5. 将任务状态置为 `ready` 或 `stale` 后再由流程模块推进。

异常流程：

| 异常 | 处理 |
|---|---|
| 工作区不可写 | 阻断加载并生成错误事件 |
| 子工作区索引损坏 | 阻断当前模式加载，提示修复或重建 |
| 任务名称重复 | 拒绝保存并提示冲突名称 |
| 输入文件不可读 | 拒绝导入，不写入任务清单 |
| 清理旧产物失败 | 标记任务 `failed`，不得生成通过结论 |

# 7. 文件、目录、数据库或配置影响

任务级目录结构：

```text
workspace/
  research_workspace/
    workspace_index.json
    mode_config.json
    system_logs/
    tasks/
      <task_id>/
        task.json
        inputs/
          program/
          program_manifest.json
          sampling_config.json
          fault_injection_config.json
        logs/
        evidence/
          artifacts.json
        waveform/
        reports/
  test_workspace/
    ...
```

`task.json` 保存任务 ID、任务名称、模式、状态、创建时间、更新时间、当前输入摘要、当前阶段状态和最近错误 ID。

数据库关联：若实现数据库，M02 对应任务表、任务输入表和任务产物索引表；所有记录以 `task_id` 为核心关联键。

# 8. 错误处理、日志与证据规则

- 任务目录、索引和输入清单写入失败均为阻断错误。
- 任务输入不得记录导入前外部路径，只记录任务内副本相对路径、原始文件名、摘要、大小和导入时间。
- 任务产物必须记录相对当前任务目录的路径。
- 旧输入对应的旧产物被清理后，不得继续作为报告通过依据。
- 有未修复的阻断错误时，不允许忽略后继续形成成功状态。

# 9. 与 UI 的数据合同

UI 可读取：

- 当前工作区显示名称，不含完整路径。
- 当前模式。
- 当前任务 ID、任务名称、任务状态。
- 当前输入摘要。
- 当前阶段进度。
- 最近错误摘要。
- 任务内报告和证据列表。

UI 必须提交：

- `mode`
- `task_id`
- `task_name`
- 输入变更动作：`overwrite_current_task`、`create_new_task`、`cancel`
- 程序、采样、故障注入等输入内容或导入请求

UI 不提交、不显示：

- 固化资源物理路径。
- 板卡 profile 物理路径。
- 报告模板物理路径。
- 安装包内部调试后端路径。

# 10. 测试场景与验收标准

| 场景 | 验收标准 |
|---|---|
| 新建工作区 | 自动创建研发/测试子工作区和基础索引 |
| 新建任务 | 生成唯一任务 ID 和任务目录 |
| 重名任务 | 保存被拒绝并提示冲突 |
| 导入程序 | 程序复制到 `inputs/program/`，清单不含外部路径 |
| 覆盖程序输入 | 清理旧日志、旧证据、旧波形和旧报告 |
| 新建任务承接变更 | 旧任务不变，新任务重新形成证据链 |
| 模式隔离 | 研发任务不可引用测试任务证据，反之亦然 |
| UI 合同 | 不出现独立历史/资源/系统状态主入口，顶部不显示工作区路径 |

# 11. 设计假设

1. 任务是用户可见、可复核、可报告的最小业务归属单位。
2. 子工作区内任务名称唯一即可，不要求项目级全局唯一。
3. 任务目录使用相对路径写入索引和报告。
4. 工作区索引损坏时，v1 先阻断并提示，不自动重建。
