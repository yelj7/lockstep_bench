# 1. 设计定位

M06 是上位机访问目标芯片的统一调试访问边界，为 M05 目标连接、M07 烧写回读运行控制以及后续采集控制提供可替换的调试命令接口。最终调试后端为自研片上调试服务，第三方工具概念不得泄漏到任务、报告、工作区索引和跨模式配置。

# 2. 设计输入与约束

- 输入来自 M03 解析后的调试适配资源和板卡 profile。
- 调用方只使用自研调试访问语义接口。
- 任务证据只保存真实返回、摘要、命令语义和资源摘要，不保存临时后端物理路径。
- 有未修复的调试访问错误时，依赖流程不得忽略后继续。
- 运行和中止的“返回确认”只确认调试命令被后端接受、目标状态可查询或形成快照；在不知道程序业务功能、预期输出、结果寄存器或协议返回的情况下，不推断程序功能正确。
- 当前 UI 和冒烟测试通过 `DebugServiceAccess` 调用自研调试服务请求/响应协议；真实板卡访问必须由自研调试服务内部传输层完成。
- `InMemoryDebugAccess` 仅允许作为单元测试或局部算法验证辅助，不得作为 UI 默认后端，不得生成真实上板通过结论。

# 3. 模块职责与不负责事项

职责：

- 建立和关闭调试访问连接。
- 查询目标状态。
- 读写目标地址空间。
- 执行复位、运行、中止。
- 保存原始返回或原始返回快照。
- 将后端错误转换为结构化错误事件。

不负责：

| 事项 | 归属 |
|---|---|
| profile schema | M03 |
| 连接业务结论 | M05 |
| 镜像解析和回读比较 | M07 |
| 报告结论 | M13 |

# 4. 数据对象与状态模型

| 对象 | 说明 |
|---|---|
| `DebugAccessContext` | task_id、mode、request_id、resource_snapshot |
| `DebugCommandRequest` | 语义命令、参数、超时、期望返回类型 |
| `DebugCommandResult` | 状态、原始返回、摘要、耗时、错误 |
| `RawReturnArtifact` | 保存到任务证据目录的原始返回或快照 |

连接状态：`closed`、`opening`、`open`、`failed`、`closing`。

命令状态：`queued`、`running`、`succeeded`、`failed`、`timeout`、`aborted`。

# 5. 对外接口与调用关系

统一接口：

| 接口 | 说明 |
|---|---|
| `open(context)` | 建立调试访问连接 |
| `close(context)` | 关闭连接 |
| `status(context)` | 查询目标状态 |
| `read(address, length, options)` | 读取目标地址空间 |
| `write(address, data, options)` | 写入目标地址空间 |
| `reset(strategy)` | 执行复位策略 |
| `run(entry_address, options)` | 启动运行 |
| `halt(options)` | 中止或暂停目标 |
| `saveRawReturn(result)` | 保存原始返回 |

调用关系：M05 负责连接和预检；M07 负责烧写、回读和运行控制；M06 只执行调试访问命令。

C++ 当前接口映射：`open/close/status/read/write/reset/run/halt` 在源码中分别由 `DebugAccess::connectTarget`、`disconnectTarget`、`status`、`read`、`write`、`reset`、`run`、`halt` 承载。UI 和 M07 只持有 `DebugAccess` 抽象；当前运行时通过 `DebugServiceAccess` 启动 exe 同级的 `lockstep_debug_service` 并传入 JSON 请求文件。后续真实板卡传输层必须在自研服务内部实现，不得要求 UI 或 M07 直接感知服务进程、驱动路径或底层实现细节。调试接口配置和目标配置使用自研 JSON schema：`lockstep-debug-interface-v1`、`lockstep-debug-target-v1`。

自研服务响应字段：

| 字段 | 说明 |
|---|---|
| `success` | 命令是否真实执行成功 |
| `error_code` | 结构化错误码，如 `TRANSPORT_NOT_IMPLEMENTED`、`INVALID_CONFIG` |
| `error_message` | 中文错误摘要 |
| `data_hex` | read 命令返回数据，十六进制编码 |
| `snapshot` | 目标状态、地址、长度、传输层状态等快照 |
| `profile` | 本次请求的 profile 摘要 |
| `config` | 本次请求的配置存在性和速度摘要 |

# 6. 主流程与异常流程

命令执行流程：

1. 调用方提交语义命令。
2. M06 校验连接状态和参数。
3. 选择当前调试适配后端。
4. 执行命令并收集真实返回。
5. 生成 `DebugCommandResult`。
6. 保存原始返回或快照到当前任务证据。
7. 返回结构化结果给调用方。

运行/中止确认口径：

1. `run` 成功条件是后端返回成功，并且至少保存一项真实返回、状态快照或目标状态查询结果。
2. `halt` 成功条件是后端返回成功，并且状态快照能够说明目标已中止、暂停或进入可控状态。
3. 若目标程序没有约定输出，M06 不判断“程序是否跑对”，只判断运行/中止控制命令是否被真实执行并留下证据。
4. 若后续需要确认程序业务结果，应由结果寄存器、串口输出、采集文件、协议解析或报告模块提供证据，不由 M06 自行猜测。

异常流程：

| 异常 | 处理 |
|---|---|
| 后端进程启动失败 | 返回连接失败错误 |
| 自研服务传输层未接入 | 返回 `TRANSPORT_NOT_IMPLEMENTED` 并阻断后续依赖阶段 |
| 命令超时 | 保存超时上下文并返回 failed/timeout |
| 原始返回为空 | 不得推断成功 |
| 原始返回保存失败 | 命令结果不得作为 pass 强制证据 |
| 用户中止 | 返回 aborted 并保存已收到返回 |

# 8. 错误处理、日志与证据规则

- M06 不吞错，不把后端异常伪造成成功。
- 未修复的连接、读写、复位、运行错误阻断依赖流程。
- warning 可记录，不阻断 pass。
- 运行控制真实返回或快照是报告 pass 的强制证据之一。
- 临时后端名称和物理路径不进入任务索引和报告正文。

# 9. 与 UI 的数据合同

UI 通过 M05/M07 查看 M06 结果摘要，不直接操作后端路径。可显示：

- 连接状态摘要。
- 最近命令语义和结果。
- 运行真实返回或快照摘要。
- 错误提示。

# 10. 测试场景与验收标准

| 场景 | 验收标准 |
|---|---|
| status 成功 | 返回目标状态并保存原始返回 |
| read/write 成功 | 返回字节数和真实返回 |
| run 成功 | 形成运行控制真实返回或快照 |
| halt 成功 | 形成中止控制真实返回或快照 |
| 命令超时 | 有错误事件和原始上下文 |
| 后端路径变化 | 任务和报告不受字段语义影响 |
| 自研服务协议联调 | UI 能发出标准请求，服务能校验 JSON 配置并返回标准响应；未接传输层时不得标记为真实上板通过 |
| 真实服务上板 | 连接、读写、复位、运行、中止均有真实返回和证据文件 |

# 11. 设计假设

1. 后端适配能力声明由 M03 资源摘要提供。
2. `request_id` 是 M06 命令追踪主键。
3. 原始返回可用 JSON 文本保存；二进制返回按文件摘要引用。
4. 真实板卡传输层由自研片上调试服务承载；任务、报告或跨模式配置语义中不得出现第三方调试工具概念。

# 12. 待用户裁决问题

无。后端不泄漏、错误不可忽略和原始返回保存口径已按用户裁决固化。
