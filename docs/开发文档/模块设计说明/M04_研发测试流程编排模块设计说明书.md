# 1. 设计定位

M04 负责把当前任务按研发模式或测试模式组织成受控流程，维护阶段状态、前置条件、失败中止和 UI 操作可用性。M04 不直接执行硬件命令，而是调度 M05、M07、M09-M12、M13 等模块，并把阶段事实写回当前任务。

# 2. 设计输入与约束

- 研发模式流程：任务、板卡连接、烧写回读、采集配置、运行控制、波形查看、协议解析、报告。
- 测试模式流程：任务、板卡连接、烧写回读、故障注入、采集配置、运行控制、波形查看、协议解析、报告。
- 测试模式显示故障注入入口，但故障注入非必须；未配置或跳过故障注入不得阻断程序跑通结论。
- 回读失败、不一致或缺少回读记录时，不允许启动运行。
- 报告 `pass` 只强制依赖：烧写成功记录、回读校验通过记录、运行控制真实返回或快照。
- 波形采集输出固定为当前任务 `waveform/lockstep_trace.vcd` 和 `lockstep_trace_schema.json`，协议解析输出固定为 `waveform/lockstep_trace_analysis.json`。
- M11 打开波形界面时若 analysis 缺失或过期，可请求 M12 后台解析；该动作不改变 `pass` 强制证据集合。

# 3. 模块职责与不负责事项

职责：

- 维护当前任务的流程阶段状态。
- 检查阶段前置条件并控制后续动作是否可用。
- 调度连接、烧写、回读、运行、采集、波形、协议解析和报告模块。
- 在失败时停止依赖阶段并记录阻断原因。
- 将 warning 保留为记录，不自动阻断 pass。

不负责：

| 事项 | 归属 |
|---|---|
| 任务目录和输入管理 | M02 |
| 资源解析 | M03 |
| 连接和预检细节 | M05 |
| 调试命令执行 | M06 |
| 镜像解析、烧写、回读、运行 | M07 |
| 报告结论生成 | M13 |

# 4. 数据对象与状态模型

核心对象：

| 对象 | 说明 |
|---|---|
| `FlowContext` | 当前模式、任务、资源和阶段上下文 |
| `StageState` | 单阶段状态 |
| `FlowGate` | 阶段前置条件结果 |
| `StageResult` | 单阶段事实摘要 |

阶段状态：`not_started`、`ready`、`running`、`passed`、`failed`、`blocked`、`skipped`、`warning`。

关键 gate：

| gate | 通过条件 |
|---|---|
| `connection_ready` | M05 连接和预检通过 |
| `program_written` | M07 烧写成功记录存在 |
| `readback_passed` | M07 回读校验通过记录存在 |
| `run_allowed` | `connection_ready && program_written && readback_passed` |
| `report_allowed` | 当前任务存在报告所需事实；失败任务也可生成 fail/incomplete 报告 |

# 5. 对外接口与调用关系

| 接口 | 调用方 | 输出 |
|---|---|---|
| `startFlow(task_id)` | UI | 初始化阶段状态 |
| `getStageState(task_id)` | UI | 当前阶段列表和阻断原因 |
| `canExecute(task_id, action)` | UI/业务模块 | 操作可用性 |
| `recordStageResult(task_id, stage, result)` | 业务模块 | 更新后的流程状态 |
| `abortFlow(task_id, reason)` | UI/M14 | 中止状态 |
| `requestReport(task_id)` | UI/M13 | 报告生成请求 |

# 6. 主流程与异常流程

研发主流程：

1. 加载任务并校验关键输入。
2. 执行板卡连接与预检。
3. 执行镜像校验、烧写和回读。
4. 只有回读通过后，才允许采集配置和运行控制。
5. 运行后 M09/M10 可执行采集并生成固定 VCD/schema。
6. M12 基于固定 VCD/schema 生成协议 analysis。
7. M11 打开波形界面时自动读取固定三文件；若 analysis 缺失或过期则后台请求 M12。
8. 生成报告时，VCD 和协议解析结果作为记录项，不作为 pass 阻塞项。

测试主流程：

1. 加载测试任务并校验关键输入。
2. 执行连接、烧写和回读。
3. 故障注入若已配置则执行并记录；未配置则标记 `skipped`，不阻断后续。
4. 回读通过后才允许采集配置和运行控制。
5. 运行后 M09/M10 可执行采集并生成固定 VCD/schema。
6. M12 基于固定 VCD/schema 生成协议 analysis。
7. M11 打开波形界面时自动读取固定三文件；若 analysis 缺失或过期则后台请求 M12。
8. 生成报告时，VCD 和协议解析结果作为记录项，不作为 pass 阻塞项。

异常流程：

| 异常 | 处理 |
|---|---|
| 连接失败 | 阻断烧写、回读、运行 |
| 镜像无效 | 阻断烧写 |
| 烧写失败 | 阻断回读通过和运行 |
| 回读失败或不一致 | 阻断运行，不提供强制运行入口 |
| 故障注入未配置 | 测试模式记录为 `skipped` |
| warning | 记录到日志和报告，不阻断 pass |

# 7. 文件、目录、数据库或配置影响

M04 写入当前任务：

- `task.json` 中的 `flow_state`。
- `evidence/artifacts.json` 中的阶段结果索引。
- `logs/operation.log` 中的流程调度日志。

M04 不决定波形产物文件名。波形相关模块统一使用：

```text
waveform/lockstep_trace.vcd
waveform/lockstep_trace_schema.json
waveform/lockstep_trace_analysis.json
```

示例：

```json
{
  "flow_state": {
    "connection": "passed",
    "program_write": "passed",
    "readback_verify": "passed",
    "fault_injection": "skipped",
    "run_control": "passed",
    "report": "ready"
  }
}
```

# 8. 错误处理、日志与证据规则

- M04 只记录流程层错误，不替代业务模块错误判断。
- 有未修复的阻断错误时，不允许继续依赖阶段。
- warning 允许进入报告并允许 pass。
- 回读失败是运行控制的硬阻断条件。
- VCD 波形和协议解析结果缺失只影响报告记录完整性，不阻断 pass；但 M11/M12 必须向 UI 和 M14 给出明确原因。

# 9. 与 UI 的数据合同

UI 可读取：

- 阶段状态。
- 操作按钮可用性。
- 阻断原因摘要。
- 故障注入是否已配置、已跳过或已执行。
- 报告生成可用性。

UI 不提供：

- 回读失败后的强制运行入口。
- 独立历史、资源、系统状态主入口。

# 10. 测试场景与验收标准

| 场景 | 验收标准 |
|---|---|
| 回读通过 | 运行按钮可用 |
| 回读失败 | 运行按钮不可用 |
| 测试任务无故障注入配置 | 故障注入 `skipped`，不阻断运行 |
| warning 存在 | 可生成 `pass` 报告并记录 warning |
| VCD 缺失 | 不阻断 pass，只记录未生成 |
| 协议解析缺失 | 不阻断 pass，只记录未生成 |
| analysis 缺失但 VCD 存在 | 打开波形页时 M11 可触发 M12 后台解析 |
| 新采集完成 | 固定 VCD/schema 覆盖旧文件，旧 analysis 失效并重新生成 |

# 11. 设计假设

1. 当前任务是流程状态的唯一业务归属。
2. 采集、波形、协议解析模块后续实现时遵循 M04 gate 和固定 waveform 三文件合同。
3. 用户中止后 v1 不做阶段级恢复，只能重新执行相关流程。

# 12. 待用户裁决问题

无。故障注入非必须、回读失败禁止运行、pass 强制证据集合已按用户裁决固化。
