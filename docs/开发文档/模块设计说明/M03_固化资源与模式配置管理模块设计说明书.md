# 1. 设计定位

M03 负责安装包固化资源的发现、校验、摘要生成和模式配置绑定。固化资源包括自研片上调试器资源、板卡 profile、报告模板、协议规则资源、字段映射和后续故障注入脚本入口。

核心原则：任务只引用稳定资源逻辑 ID、版本和摘要，不保存具体后端路径、模板路径或板卡 profile 物理路径。OpenOCD 若用于研发初期，只能作为 M06 内部可替换适配项，不能成为任务、报告或 UI 的固定领域概念。

# 2. 设计输入与约束

设计输入：

- 需求规格说明书、软件设计说明书、数据库设计说明书、开发计划、任务进度表。
- UI 设计说明书。
- 用户提供的 `C:\Users\zzz\Desktop\profile.json` 样例。
- 用户裁决：协议规则先作为占位；warning 允许通过；v1 报告只要求 `report.json`。

约束：

| 约束 | 设计口径 |
|---|---|
| 资源只读 | `install_root/resources/` 不写任务、证据、报告 |
| 路径隔离 | 任务差异输入不得保存具体后端路径、模板路径或 profile 物理路径 |
| 模式隔离 | 研发/测试模式各自保存模式配置摘要 |
| UI 边界 | 不提供独立资源主入口 |
| 协议资源 | v1 允许登记为 `placeholder`，不阻断程序跑通结论 |

# 3. 模块职责与不负责事项

模块职责：

- 读取并解析 `resources/manifest.json`。
- 校验固化资源 ID、版本、相对路径、摘要和启用状态。
- 校验板卡 profile 的必要字段、地址字段、调试适配字段和运行控制字段。
- 校验报告模板的模式归属、版本、摘要和必需占位。
- 为协议规则资源生成占位摘要。
- 向 M05/M06/M07/M13 提供逻辑资源解析服务。
- 将当前模式使用的资源摘要写入 `mode_config.json` 或数据库模式配置记录。

不负责事项：

| 不负责事项 | 归属模块 |
|---|---|
| 工作区和任务创建 | M02 |
| 目标连接业务结论 | M05 |
| 调试命令执行 | M06 |
| 镜像格式解析和烧写范围检查 | M07 |
| 报告结论 | M13 |
| 错误提示解码 | M14 |

板卡 profile v1 schema 参考用户样例：

```json
{
  "profileName": "NOEL-V ZCU102 CMSIS-DAP RISC-V JTAG Working",
  "host": "127.0.0.1",
  "gdbPort": 3333,
  "tclPort": 6666,
  "jtagKhz": 14,
  "interfaceConfigPath": "resources/board_profiles/noelv_zcu102/cmsis-dap-explicit.cfg",
  "targetConfigPath": "resources/board_profiles/noelv_zcu102/target-riscv-direct.cfg",
  "targetDebugToolPath": "resources/debug_tools/openocd/openocd.exe",
  "ramBaseAddress": "0x0",
  "defaultRunAddress": "0x0",
  "resetStrategy": "reset halt",
  "runAfterDownload": true
}
```

profile 路径字段允许在固化资源内部保存，但解析后对任务、报告和 UI 只暴露 `profile_id`、`profileName`、版本、摘要、地址约束和运行控制摘要。

资源状态：

| 状态 | 含义 | 是否阻断 |
|---|---|---|
| `enabled` | 已启用且摘要匹配 | 否 |
| `placeholder` | 占位资源，暂不提供完整能力 | 不阻断程序跑通结论 |
| `warning` | 非关键资源异常 | 不阻断 pass，但写入报告记录 |
| `missing` | manifest 声明但文件缺失 | 关键资源阻断 |
| `invalid` | 格式或摘要错误 | 关键资源阻断 |
| `disabled` | manifest 明确关闭 | 相关能力不可用 |

# 5. 对外接口与调用关系

| 接口 | 调用方 | 输出 |
|---|---|---|
| `validateResourcePack()` | 启动流程/M14 | 资源校验结果 |
| `getModeResourceSnapshot(mode)` | M02/M04/M13 | 当前模式资源摘要 |
| `listBoardProfiles(mode)` | UI/M05 | profile 列表 |
| `resolveBoardProfile(profile_id)` | M05/M07 | profile 内容和摘要 |
| `resolveDebugAdapter(adapter_id)` | M06 | 调试适配配置 |
| `resolveReportTemplate(template_id)` | M13 | `report.json` 模板/schema |
| `resolveProtocolRule(rule_id)` | M12/M13 | 协议规则状态和摘要 |

调用关系：启动时 M01/M02 触发资源校验；M04 在执行流程前检查关键资源状态；M05/M06/M07/M13 只通过逻辑 ID 获取运行时内容。

# 6. 主流程与异常流程

资源校验流程：

1. 定位 `install_root/resources/manifest.json`。
2. 校验 manifest JSON 结构。
3. 对每个资源项检查 `id`、`relative_path`、`sha256`、`status`。
4. 校验 profile 必填字段。
5. 校验默认 profile 和默认报告模板是否存在。
6. 生成资源摘要并写入模式配置记录。

异常流程：

| 异常 | 处理 |
|---|---|
| manifest 缺失或格式错误 | 阻断依赖资源流程 |
| 默认 profile 缺失 | 阻断连接、烧写、回读和运行 |
| profile 必填字段缺失 | 阻断连接 |
| 调试工具或配置摘要不匹配 | 阻断连接、烧写、回读和运行 |
| 报告模板缺失 | 阻断报告生成 |
| 协议规则为占位 | 允许报告记录为未启用，不阻断 pass |

# 7. 文件、目录、数据库或配置影响

安装后资源目录：

```text
install_root/
  resources/
    manifest.json
    board_profiles/
    debug_adapters/
    report_templates/
    protocol_rules/
```

模式配置摘要：

```json
{
  "resource_pack_id": "lockstep.default",
  "resource_pack_version": "1.0.0",
  "manifest_sha256": "<sha256>",
  "profile_id": "board.noelv.zcu102.cmsisdap",
  "profile_sha256": "<sha256>",
  "report_template_id": "report.research.default",
  "report_template_sha256": "<sha256>",
  "protocol_rule_id": "protocol.placeholder",
  "protocol_rule_status": "placeholder"
}
```

禁止写入任务差异输入和报告正文的内容：

- 调试后端可执行文件绝对路径。
- 报告模板物理路径。
- 板卡 profile 物理路径。
- OpenOCD 等临时后端名称作为固定领域字段。

# 8. 错误处理、日志与证据规则

M03 错误事件字段包括：`code`、`severity`、`source_module=M03_RESOURCE_CONFIG`、`failure_stage`、`resource_id`、`workspace_mode`、`blocking_flag`、`message`、`suggestion`。

错误编码无统一格式强制要求，采用适合实现的稳定编码即可；同一错误码含义在版本内必须稳定。

证据规则：

- 任务证据引用资源时只记录 ID、版本和摘要。
- `placeholder` 和 `warning` 资源状态可进入报告记录。
- 缺失或摘要不匹配的关键资源不得被伪造成通过证据。
- 资源校验日志写 `system_logs/`，不作为任务 pass 强制证据。

# 9. 与 UI 的数据合同

UI 可显示：

| 页面 | 可显示字段 |
|---|---|
| 板卡连接页 | profile 名称、版本、地址约束摘要 |
| 烧写回读页 | 默认入口和下载地址约束摘要 |
| 协议解析页 | 协议规则状态，`placeholder` 显示为占位 |
| 报告页 | 当前模式模板名称、版本、摘要 |
| 顶部状态栏 | 最近资源错误摘要 |

UI 只提交 `profile_id`、`template_id`、`rule_id` 等逻辑标识。

# 10. 测试场景与验收标准

| 场景 | 验收标准 |
|---|---|
| manifest 正常 | 校验通过并形成资源摘要 |
| manifest 缺失 | 明确报错并阻断 |
| profile 样例字段完整 | 可被解析为板卡 profile |
| profile 缺字段 | 板卡连接不可执行 |
| 模板缺失 | 报告生成不可执行 |
| 协议规则占位 | 不阻断程序跑通 pass |
| OpenOCD 适配 | 任务、报告、UI 不出现固定 OpenOCD 概念 |

# 11. 设计假设

1. `resources/manifest.json` 由打包流程生成或维护。
2. 所有 manifest 路径均相对 `resources/`。
3. profile 物理路径可在资源内部存在，但不会进入任务证据和报告正文。
4. v1 只强制 `report.json` 模板能力。
