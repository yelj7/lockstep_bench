# 1. 设计定位

M05 位于流程编排和自研片上调试器之间，负责根据当前任务、模式配置和板卡 profile 建立目标访问条件，查询目标状态，并形成“目标是否可访问、预检是否通过”的业务结论。

# 2. 设计输入与约束

输入包括当前任务、M03 解析后的 profile、调试适配资源摘要和 M06 调试访问接口。profile 参考字段包括 `profileName`、`host`、`gdbPort`、`tclPort`、`jtagKhz`、`interfaceConfigPath`、`targetConfigPath`、`targetDebugToolPath`、`ramBaseAddress`、`defaultRunAddress`、`resetStrategy`、`runAfterDownload`。

约束：

- UI 和任务不得保存 profile 物理路径。
- 连接失败或预检失败时，烧写、回读和运行均不可执行。
- 有未修复的连接错误时不能忽略后继续。

# 3. 模块职责与不负责事项

职责：

- 校验当前 profile 是否可用于当前模式。
- 请求 M06 建立调试访问连接。
- 查询目标状态、复位能力、基础读写能力和运行控制能力。
- 生成连接记录、预检记录和阻断原因。

不负责：

| 事项 | 归属 |
|---|---|
| profile schema 维护 | M03 |
| 调试命令执行 | M06 |
| 镜像解析和烧写 | M07 |
| 报告结论 | M13 |

# 4. 数据对象与状态模型

| 对象 | 说明 |
|---|---|
| `ConnectionRequest` | 任务、模式、profile_id、资源摘要 |
| `ConnectionRecord` | 连接尝试、真实返回摘要和状态 |
| `PrecheckRecord` | 预检项、结果和阻断原因 |

连接状态：`not_connected`、`connecting`、`connected`、`failed`、`disconnected`。

预检状态：`not_run`、`passed`、`failed`、`blocked`。

# 5. 对外接口与调用关系

| 接口 | 调用方 | 输出 |
|---|---|---|
| `connectTarget(task_id, profile_id)` | UI/M04 | 连接记录 |
| `disconnectTarget(task_id)` | UI/M04 | 断开记录 |
| `runPrecheck(task_id)` | M04/UI | 预检记录 |
| `getConnectionGate(task_id)` | M04/M07 | 后续流程是否允许 |
| `getLastTargetStatus(task_id)` | UI | 最近状态摘要 |

# 6. 主流程与异常流程

主流程：

1. 根据 `profile_id` 从 M03 获取 profile 内容和摘要。
2. 校验 profile 必填字段和资源摘要。
3. 调用 M06 连接目标。
4. 查询状态、复位能力、读写能力和运行控制能力。
5. 保存连接真实返回或原始返回摘要。
6. 输出 `connection_ready=true/false`。

异常流程：

| 异常 | 处理 |
|---|---|
| profile 缺失 | 阻断连接 |
| profile 摘要不匹配 | 阻断连接并记录差异 |
| 调试适配启动失败 | 阻断连接 |
| 目标状态不可读 | 预检失败 |
| 原始返回保存失败 | 不得标记连接通过 |

# 7. 文件、目录、数据库或配置影响

M05 写入当前任务：

- `logs/operation.log`
- `evidence/artifacts.json`
- `evidence/connection_record.json`
- `evidence/precheck_record.json`

记录只保存 `profile_id`、profile 摘要和能力摘要，不保存本机绝对路径。

# 8. 错误处理、日志与证据规则

- 连接失败、预检失败和原始返回缺失是阻断错误。
- 阻断错误未修复时不能忽略后继续。
- warning 可记录为非阻断提示。
- M05 不为报告生成 pass，只提供连接和预检事实。

# 9. 与 UI 的数据合同

UI 显示字段：

- profile 名称和摘要。
- 连接状态。
- 预检状态。
- 最近失败原因。
- 后续操作 gate。

UI 不显示：

- `targetDebugToolPath`
- `interfaceConfigPath`
- `targetConfigPath`
- 安装包或用户机器绝对路径。

# 10. 测试场景与验收标准

| 场景 | 验收标准 |
|---|---|
| profile 完整 | 可发起连接 |
| profile 缺路径字段 | 连接被阻断 |
| 目标无响应 | 预检失败并记录真实返回 |
| 预检通过 | M07 可执行烧写 |
| 预检失败 | M07 烧写、回读、运行不可执行 |

# 11. 设计假设

1. v1 使用单目标连接模型。
2. 连接超时和重试策略由实现层选择合适默认值，可后续移入 profile。
3. 非破坏诊断命令若未来需要，由 M06 提供受控接口。

# 12. 待用户裁决问题

无。profile 参考样例和不可忽略阻断错误口径已按用户裁决固化。
