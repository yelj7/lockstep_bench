# NOEL-V ZCU102 锁步采集覆盖层

本目录是 `lockstep_bench` 对 NOEL-V/GRLIB 基线的项目自有维护层，不包含完整上游源码。

构建输入由两部分组成：

1. 固定的 NOEL-V/GRLIB 基线树，其文件摘要由 `manifests/baseline_source_manifest.txt` 锁定。
2. 本目录下的采集 RTL、NOEL-V 集成文件、约束和构建脚本。

构建时必须把 `integration/` 和 `rtl/` 中的相对路径覆盖到基线树，再使用 `build/` 中的脚本。不得直接从历史 `.codex-tasks` 快照构建发布产物。

## 门禁

- 本地只允许静态检查和不调用 Vivado 的软件测试。
- XSim、综合、实现和 bitstream 只允许在 `ylj@192.168.31.78` 运行。
- 顶层综合必须为非 OOC。
- 除已确认的 Xilinx 标准调试基础设施外，不允许黑盒。
- debug/release 均要求无未路由网络、DRC 无错误、WNS/WHS 非负。
- 每次构建必须归档 source manifest、仿真报告、时序/DRC/路由报告和 `SHA256SUMS`。

## 协议版本

- v2 连续窗口帧保持兼容：`4096 x 1024-bit`。
- v3 新增独立的长时间协议事件流；事件流不得改变连续窗口的采样语义。
- ARM ACK 后主机用 `START_EVENT_STREAM` 释放事件上传，EVENT/SAMPLE 按完整帧轮询。
- 事件使用扩展后的 64-bit 全局样本索引；当前 `program_done` 尚无真实顶层来源。
- ETH/USB 在真实探针源接入前必须报告 `unavailable/design_gap`，不得生成伪事务。
